# Contributing

欢迎贡献 issue、案例、模板改进、工具集成方式和实践反馈。

## 1. 可以贡献什么

- 新案例。
- 更好的 AGENTS.md 模板。
- 更好的 CommandPack 模板。
- 更好的 ContextPack 模板。
- 不同技术栈的示例项目。
- UTF-8 / 编码检查脚本改进。
- CI / Git Hook 集成示例。
- 文档错别字和结构优化。
- 英文翻译。

## 2. 贡献原则

请遵守本项目核心理念：

> 人负责方向、边界、判断和验收；  
> AI 负责在明确任务合同下执行；  
> 仓库文档负责保存长期上下文；  
> 测试和门禁负责验证结果。

不要把本项目改成“全自动开发神器”宣传口径。

## 3. 提交案例时的隐私要求

真实案例必须移除：

- 密钥。
- 用户数据。
- 客户名称。
- 公司内部接口。
- 私有业务规则。
- 不应公开的商业信息。

如果不确定是否可以公开，请提交抽象化案例。

## 4. PR 要求

PR 应包含：

- 目标。
- 修改文件。
- 变更说明。
- 是否涉及中文文件。
- 是否运行 UTF-8 检查。
- 是否新增脚本或模板。
- 兼容性风险。

建议运行：

```bash
python scripts/check_utf8.py .
python scripts/validate_project_structure.py .
```

## 5. 文档风格

- 中文优先，表达清楚。
- 少用空泛口号，多给可执行规则。
- 模板要能复制粘贴使用。
- 规则要有 STOP 条件、门禁和回滚意识。
