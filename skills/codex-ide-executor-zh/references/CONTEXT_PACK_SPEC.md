# ContextPack 规范

ContextPack 是仓库中给 AI 快速读取的项目上下文包。

它不是所有文档的复制，而是当前项目状态的压缩版。

---

## 1. 三层结构

### L1：最小可用上下文

用于 FAST / 小 PR。

包含：

- 项目定位。
- 当前版本。
- 主要目录。
- 常用命令。
- 当前重点。
- 禁止事项。
- 关键模块入口。

### L2：深入协作上下文

用于 SAFE / 跨模块任务。

包含：

- 主要业务流程。
- 数据结构。
- 模块依赖关系。
- 已知技术债。
- 当前架构边界。
- 最近 PR 总结。

### L3：审计交接上下文

用于 AUDIT / 重大重构 / 迁移 / 发布前检查。

包含：

- 完整架构说明。
- 历史决策。
- 风险点。
- 回滚策略。
- 已完成阶段。
- 未完成阶段。
- 下一阶段计划。

---

## 2. 更新时机

以下情况必须更新 ContextPack：

- 完成重要 PR。
- 新增模块。
- 改变数据结构。
- 改变测试命令。
- 改变项目阶段。
- 修复关键 bug。
- 调整下一阶段计划。
- 改变模块边界。

---

## 3. 禁止事项

不要把 ContextPack 变成所有文档的复制。

不要写：

- 长篇代码解释。
- 过期计划。
- 未确认想法。
- 与当前状态无关的聊天记录。
- 不可公开的密钥或业务数据。
