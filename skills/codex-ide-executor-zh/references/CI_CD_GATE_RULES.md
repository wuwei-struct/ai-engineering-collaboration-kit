# CI/CD 门禁规则

## 1. CI 是项目事实的一部分

如果仓库配置了强制 CI 门禁，应把它视为项目事实。

常见门禁：

- lint
- typecheck
- test
- build
- coverage
- contentCheck
- UTF-8 check

---

## 2. 自动升级

如果仓库配置强制 CI：

- FAST MODE 自动升级 SAFE MODE。
- 涉及发布、生产、权限、license、schema 的任务升级 AUDIT MODE。

---

## 3. 失败处理

如果本地门禁或 CI 失败，不得声称任务完成。

必须说明：

- 失败命令。
- 失败摘要。
- 是否与本次改动相关。
- 是否属于历史遗留失败。
- 当前是否阻断合并。
- 建议修复路径。

---

## 4. 可接受的跳过

可以说明并跳过：

- 本地缺少外部服务。
- 当前环境无法运行 GUI / Electron 打包。
- 需要用户本机凭证的真实接口测试。
- 任务明确不涉及相关模块。

不可隐式跳过：

- UTF-8 检查。
- 安全检查。
- 与本次改动直接相关的测试。
- 强制 CI。
