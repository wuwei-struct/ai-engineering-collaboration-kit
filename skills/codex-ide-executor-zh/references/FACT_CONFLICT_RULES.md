# 仓库事实冲突处理规则

仓库事实可能相互矛盾，例如 README 过时、测试过时、代码已经迁移、AGENTS.md 没更新。

不得基于单一老旧文档直接做决定。

---

## 1. 事实优先级

一般按以下顺序判断当前事实：

1. 当前实际代码实现。
2. 最近仍在通过的测试。
3. 最近的 `AGENTS.md` 或当前目录下更近的项目规则。
4. 最近的 ContextPack / AI_HANDOFF / PR_SUMMARIES。
5. 架构文档 / 模块边界文档。
6. README。
7. 旧文档、聊天上下文、Memory、外部工具结果。

---

## 2. 冲突处理

代码与文档冲突：

- 不要直接按旧文档重写代码。
- 先判断任务是改代码还是更新文档。
- 无法判断则 STOP。

测试与代码冲突：

- 如果测试仍是项目门禁，优先尊重测试。
- 如果测试明显过时，报告，不擅自删除或大改测试。
- 如果本次任务就是更新行为，才同步更新相关测试。

AGENTS.md 与当前任务冲突：

- 普通工程规则可由明确任务覆盖，但报告中说明。
- 安全、密钥、生产配置、数据破坏、权限、license、计费、UTF-8 不能被隐式覆盖。

---

## 3. 报告格式

```md
## 仓库事实冲突

- 冲突文件 / 规则：
- 冲突内容：
- 本次采用的事实来源：
- 采用原因：
- 是否需要后续更新文档：
```
